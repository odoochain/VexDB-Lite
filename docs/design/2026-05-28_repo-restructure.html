4. 关键不变性(重构后必须守住)

  1. common/ 不依赖任一后端:它是叶子,只 include thirdparties/ 和自己。任何 #include "../vexdb-pg/..." 在 common/ 里都是违例
  2. vexdb-pg/ 不依赖 vexdb-duck/,反之亦然:两后端通过 common/ 间接共用代码,绝不互相 reach
  3. common/include/ 里的 header 不得 #include "postgres.h":否则 vexdb-duck 编译被污染
  4. vexdb-pg/CMakeLists.txt 自包含:不再有"根目录 CMakeLists 编 PG"的隐式结构,cd vexdb-pg && cmake . 可直接 build
  5. vexdb-duck/CMakeLists.txt 路径全部从 ../src/distance/... 改成 ../common/distance/src/...:切断对根 src/ 的引用
  6. 顶层 build.sh / scripts/release.sh 是薄壳:只 dispatch 到子项目,不再自带编译细节

目录/项 候选归属 说明
